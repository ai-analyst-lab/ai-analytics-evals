# Data governance (the floor)

This repo is public. The rule that makes "open everything" safe is that no credential and no
private answer key ever reaches the public side, and every dataset we ship is one we are
allowed to ship.

## Credentials

Live credentials live only in `ai-analyst-plus/.env`, which is gitignored and never committed.
This repo declares no warehouse drivers and holds no credentials. The secret scan
(`tools/secret_scan.py`) runs before any public push and fails on any password pattern, private
key, or tracked `.env`. Run it: ask Claude to "run the secret scan," or `python3
tools/secret_scan.py`.

## Datasets

Synthetic or licensed only. Each dataset we ship is either generated by us (synthetic, no real
people) or carries a license that permits redistribution, and each gets a short note recording
which. No dataset with real personal data ships without de-identification.

- NovaMart: synthetic. No real people. Safe to ship.
- Strava (when added): personal activity data. De-identify before it ships (no names, no exact
  home locations, no raw GPS start points).
- Any public benchmark (BIRD, Spider, Cookie Cats, Olist) ships only by reference or under its
  stated license, never re-hosted if the license forbids it.

## Answer keys

Our private reference answer keys start local and stay off the public side (a stale shell or a
stray commit must not expose them). The secret scan and `.gitignore` are the guard.
